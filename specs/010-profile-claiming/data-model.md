## Entity Overview

This feature introduces **one new model** and modifies the existing `Person` and allauth adapter layer. No changes to the core data model backbone (Project, Dataset, Sample, Measurement) are required.

Claim tokens are **stateless** — generated via `TimestampSigner` and require no database table. The signed token encodes the Person PK and a timestamp; expiry and authenticity are enforced cryptographically. When a token is consumed, the event is recorded in `ClaimingAuditLog`.

Fuzzy name matching (US5) is computed **on-demand** in the admin — no persistence needed for a first implementation.

```
Person (existing — AUTH_USER_MODEL)
  └── ClaimingAuditLog (new — immutable audit trail)
```

## Stateless Claim Tokens

**No database model.** Claim tokens are signed strings generated by Django's `TimestampSigner`:

```python
# Token = signed Person PK (HMAC-SHA256 + timestamp baked in)
signer = TimestampSigner(salt="fairdm.contributor.claim")
token_string = signer.sign(str(person.pk))

# Validation — decodes PK and checks age in one call
pk = signer.unsign(token_string, max_age=7 * 24 * 60 * 60)  # raises on tamper/expiry
person = Person.objects.get(pk=pk, is_claimed=False)  # raises if already claimed
```

**Properties:**

- Token encodes: Person PK + signing timestamp
- Expiry: cryptographically enforced (not a DB field) — default 7 days via `CLAIM_TOKEN_MAX_AGE`
- Revocation: implicit — once the Person is claimed (`is_claimed=True`), the token's `Person.objects.get(is_claimed=False)` lookup fails, making the token inert
- No DB cleanup job needed; `ClaimingAuditLog` records usage when the token is consumed

## New Models

### ClaimingAuditLog

**Purpose:** Immutable record of all claiming events for security auditing and compliance. Records are never modified or deleted.

**Location:** `fairdm/contrib/contributors/models.py`

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | AutoField | PK | Primary key |
| `timestamp` | DateTimeField | auto_now_add | When the event occurred |
| `method` | CharField(20) | choices, required | Claiming method used |
| `source_person` | ForeignKey(Person) | on_delete=SET_NULL, null=True | The unclaimed Person being claimed |
| `target_person` | ForeignKey(Person) | on_delete=SET_NULL, null=True | The resulting claimed Person |
| `initiated_by` | ForeignKey(Person) | on_delete=SET_NULL, null=True, blank=True | Admin who initiated (if admin-driven) |
| `ip_address` | GenericIPAddressField | null=True, blank=True | Request IP for security auditing |
| `success` | BooleanField | required | Whether the claim succeeded |
| `failure_reason` | CharField(255) | blank=True, default="" | Reason for failure (if success=False) |
| `details` | JSONField | default=dict | Additional structured metadata |

**Choices:**

```python
class ClaimMethod(models.TextChoices):
    ORCID = "orcid", _("ORCID Social Login")
    EMAIL = "email", _("Email Verification")
    TOKEN = "token", _("Claim Token Link")
    ADMIN_MERGE = "admin_merge", _("Admin-Initiated Merge")
    ADMIN_MANUAL = "admin_manual", _("Admin Manual Activation")
```

**Constraints:**

- No update or delete operations permitted at application layer (enforced via model `save()` override that blocks updates on existing records)
- `details` JSONField stores method-specific metadata (e.g., token ID for token claims, ORCID for ORCID claims)

**Manager Methods:**

```python
ClaimingAuditLog.objects.for_person(pk)     # source_person=pk OR target_person=pk
ClaimingAuditLog.objects.failures()         # success=False
ClaimingAuditLog.objects.by_method("orcid") # Filter by claiming method
ClaimingAuditLog.objects.recent(days=30)    # Last N days
```

**Immutability enforcement:**

```python
def save(self, *args, **kwargs):
    if self.pk:
        raise ValueError("ClaimingAuditLog records are immutable and cannot be updated.")
    super().save(*args, **kwargs)
```

## Settings

All claiming methods are enabled by default with no per-method toggles. The only configurable settings are for claim token behaviour:

### CLAIM_TOKEN_MAX_AGE

```python
# Default: 7 days in seconds. Override in portal settings to adjust token expiry.
CLAIM_TOKEN_MAX_AGE = 7 * 24 * 60 * 60
```

### CLAIM_TOKEN_SALT

```python
# HMAC salt. Should not need overriding, but can be changed to invalidate all outstanding tokens.
CLAIM_TOKEN_SALT = "fairdm.contributor.claim"
```

```
### ClaimToken (Stateless — no state machine)

Tokens are opaque signed strings. Validity is determined at use-time:
- HMAC verification fails → invalid (tampered)
- Timestamp age > CLAIM_TOKEN_MAX_AGE → expired
- `Person.objects.get(pk=pk, is_claimed=False)` fails → already claimed (implicit revocation)
```

### ClaimToken State Machine

```
    ┌─────────┐   Admin generates    ┌─────────┐
    │ (none)  │ ──────────────────► │ Pending │
    └─────────┘                     └────┬────┘
                                         │
                          ┌──────────────┼──────────────┐
                          │              │              │
                    ┌─────▼─────┐  ┌────▼────┐  ┌─────▼─────┐
                    │  Claimed  │  │ Expired │  │  Revoked  │
                    │ (used ok) │  │ (7-day) │  │ (admin)   │
                    └───────────┘  └─────────┘  └───────────┘
```
